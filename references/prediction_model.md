# 账号公开互动基础盘模型

## 当前落地范围

当前只落地第一项：账号公开互动基础盘。

本项不判断转化率、ROI、产品适配度或达人长期价值，只判断账号公开互动底座是否稳定、四值结构是什么、平均值是否有代表性。

播放量不进入第一项表格、汇总和判断。抖音公开侧播放量经常不可用，优先使用点赞、评论、收藏、分享构建公开互动画像。

## 三个子模块

| 子模块 | 看什么 | 输出什么 |
|---|---|---|
| 初步账号判断 | 互动档位、均值代表性、结构倾向、稳定性 | 稳定型/波动型、是否被爆款拉高、互动结构初判 |
| 互动规模 | 近N条总互动平均、中位、最高、最低 | 基础互动档位 |
| 四值结构 | 点赞 : 评论 : 收藏 : 分享 | 结构类型和结构判断，结构判断必须写明点赞数据 |
| 均值代表性 | 平均/中位、最高/中位 | 平均值是否可信 |
| 置顶排除 | 主页前排置顶/疑似置顶作品 | 当前账号常态样本 |

## 四值结构

默认输出两种结构：

| 结构 | 示例 | 用途 |
|---|---|
| 原始结构 | `216 : 94 : 68 : 169` | 真实数量感 |
| 标准化结构 | `1 : 0.44 : 0.31 : 0.78` | 人工横向比较 |

不要默认输出 `评论/点赞`、`收藏/点赞`、`分享/点赞` 这类两两比值；它们会把四值结构拆碎。标准化结构只是为了看四个值的形态。

结构判断不是只看评论、收藏、分享。点赞也是采集字段，必须先写明“点赞X作为基准”，再判断评论、收藏、分享相对这个点赞基准是否突出。

## 均值代表性

| 条件 | 判断 |
|---|---|---|
| 样本少于5条 | 样本不足 |
| 平均/中位 <= 1.3 且最高/中位 <= 2 | 高代表性 |
| 平均/中位 <= 2 且最高/中位 <= 5 | 中代表性 |
| 平均/中位 > 2 或最高/中位 > 5 | 低代表性 |

低代表性说明平均值可能被一两条高互动作品拉高，判断账号常态时应优先看中位数。

## 置顶作品处理

置顶作品通常是达人主动挑出来放在主页顶部的历史高光内容，不能代表账号当前常态。

处理规则：

1. `creator_posts.csv` 不写入置顶/疑似置顶作品。
2. 基础盘汇总表默认排除置顶/疑似置顶作品。
3. `--count 10` 指最终进入基础盘的 10 条非置顶普通作品，不是主页前 10 条。
4. 如果排除置顶后普通作品不足目标数，继续向后翻页补抓，直到补足或账号无更多作品。
5. 如果接口或页面直接返回 `置顶`、`is_top`、`is_pinned` 等标记，记为 `is_pinned=yes`。
6. 如果主页前 3 条作品的发布时间明显早于后续更新作品，说明它违背正常时间倒序，记为 `is_pinned=likely`。
7. 单条视频链接没有主页上下文时，不判断置顶，只标记为 `unknown`。
8. 置顶/疑似置顶作品只允许在 `raw.json` 留存为排除证据，不进入给用户看的作品明细表。

## 输出口径

第一项先分样本池，再决定输出文件：

1. 有非置顶、非挂车普通作品时，输出普通作品明细、普通作品汇总表和普通作品判断报告。
2. 有挂车作品时，输出挂车作品明细、挂车作品汇总表和挂车作品判断报告。
3. 如果全是挂车作品，不生成普通作品判断报告；不要用挂车样本去填普通作品基础盘。
4. 如果普通作品和挂车作品都存在，两套分开输出，不混算。
5. `raw.json` 必须记录 `analysis_route.mode`，让 2A/2B 知道本次是 `normal_only`、`cart_only`、`mixed` 还是 `empty`。

## 初步账号判断边界

MD 必须先给一个初步账号判断，不能只说“可能需要进一步判断”。判断可以包括：

1. 账号公开互动基础盘是稳定型、有波动，还是爆款拉高型。
2. 评论是否是主要优势，账号是不是强讨论型。
3. 收藏/分享是否相对突出，是否体现出一定保留或转发动机。
4. 当前是否值得进入下一轮评论内容、商品相关作品、报价和合作链路分析。

但这些判断只能基于公开互动基础盘，不能直接写成转化率、ROI、合作价值或达人长期价值结论。

## 风险提示

样本少于 5 条时，不要判断账号常态。

本项只看公开互动，不代表粉丝信任深度。评论内容分析属于后续维度，不在本项里完成。
