# 账号内容资产盘模型

## 当前落地范围

第二项落地为：账号内容资产盘，分为 2A、2B 和 2C。

本项回答三个问题：

1. 账号主要靠什么内容获得互动。
2. 内容和母婴、儿童、营养、健康、育儿焦虑是否有自然连接。
3. 是否值得进入下一轮评论信任盘和商品/合作适配判断。

本项不默认进入具体产品承接，不写某个具体产品怎么接。2C 单独做“优质内容到营养品的转接预判”，判断账号已经跑出来的优质内容结构，能不能自然改写成儿童营养品议题。

## 查看方式

先按 `raw.json` 里的 `analysis_route.mode` 判断样本池：普通作品池、挂车作品池，或两者都有。再用对应样本池做 2A 粗筛，抽样看真实媒体内容做 2B 细看，最后基于 2A + 2B 做 2C 转接预判。不要把 2A 当成完整内容判断，也不要在没有看真实媒体内容时进入 2C。

样本池路由：

- `normal_only`：只生成普通作品 2A/2B/2C。
- `cart_only`：只生成挂车作品 2A/2B/2C，不生成普通作品分析文档。
- `mixed`：普通作品和挂车作品分开分析，不能混算。
- `empty`：没有可分析样本，不生成正式 2A/2B/2C。

硬门槛：

- 视频作品必须查看具体抽帧图片 `2b_<作品ID>_grid.jpg`，不是只看视频标题、互动数据、下载状态或视频链接。
- 图片/图文作品必须查看具体图片文件或图片清单。
- 没有具体媒体证据时，只能生成待补看清单，不能生成 2B 正式细看，也不能继续生成 2C。
- 生成 2B 抽样清单后，必须先询问用户是否下载媒体。用户可以按系统抽样下载，也可以自行提供想详细拆解的视频/图文链接，或指定清单里的某几条。
- 下载后继续抽帧/读取图片、交给媒体理解 Agent 看图/看视频/转写，都必须先询问。2C 不需要单独询问，但必须建立在 2B 已看具体媒体证据之后。

## 2A 标题/话题/挂车粗筛

粗筛字段：

| 字段 | 用途 |
|---|---|
| 作品标题/描述 | 初步识别主题、对象、场景和商品内容信号 |
| 标题话题 `#...` | 做商品仲裁：识别品牌词、商品词、品类词、活动词 |
| 挂车/商品锚点字段 | 如果公开接口返回 `anchors`、商品锚点或电商字段，作为强商品信号证据 |
| 点赞/评论/收藏/分享 | 判断不同主题的互动表现 |
| 结构类型/结构判断 | 判断主题对应的互动动机 |
| 发布时间 | 判断内容是否是近期常态 |
| 原始接口字段 | 查找封面、话题、作者信息、商品/挑战/贴纸等辅助线索 |

抽样优先级：

1. 高互动作品：看账号最能打的内容靠什么起量。
2. 中位互动作品：看账号常态内容是什么。
3. 低互动作品：看什么内容不吃香。
4. 商品相关作品：看粉丝对种草内容、商品推荐内容的接受度。
5. 品类相关作品：看是否存在母婴营养品方向的自然入口。

置顶/疑似置顶视频可以作为“主页展示资产”单独看，但不能混入普通作品常态判断。

## 2B 媒体内容细看

2B 必须实际看媒体内容，不能只看标题、话题和挂车字段。样本可能是视频，也可能是图片/图文。视频样本必须看具体抽帧图片；如果没有 `2b_<作品ID>_grid.jpg`，这条样本只能进入待补看清单，不得写正式判断。

2B 的抽样和拆解不能按“总互动”一把抓。必须先把当前样本池里的作品分别按点赞、评论、收藏、分享四项互动排序，找出每一项的高位样本和低位样本，再合并抽样。全挂车时，当前样本池就是挂车作品池，不要强行寻找普通主页作品。

抽样规则：

1. 分别找点赞高位 / 低位、评论高位 / 低位、收藏高位 / 低位、分享高位 / 低位。
2. 如果一条视频同时命中多个高位或低位锚点，只抽一次，但在抽样理由里写清它覆盖哪些锚点。
3. 中位样本只作为常态对照，不要替代四项高低位样本。
4. 强品类连接但四项互动低的样本必须保留，因为它能验证“内容相关但账号粉丝不响应”。
5. 不得只因为总互动最高就抽样；总互动只能辅助发现异常样本，不能作为 2B 抽样主逻辑。

2B 拆媒体时，必须把点赞、评论、收藏、分享当作四个不同动作来解释：

| 互动锚点 | 2B 要回答 |
|---|---|
| 点赞 | 内容靠什么获得认可、共鸣、真实感或价值认同 |
| 评论 | 内容靠什么引发提问、争议、经验交换、求链接或求型号 |
| 收藏 | 内容靠什么形成复看价值、清单价值、步骤价值或购买前参考价值 |
| 分享 | 内容靠什么值得转给家人、朋友、同类家长或共同决策人 |

视频作品的前三秒技法在 2B 里只作为判断维度，不写成外部方法论硬调用。执行时按以下顺序判断：

1. 先判断不划走理由：异常的侦测、对自我的确认、对威胁的规避、对同类的共情、对未知的探求、对奖赏的预期、对群体的归属。
2. 再判断前三秒文案类型：身份推荐、点名人群、直陈痛点、直陈效果、提出疑问、引发好奇、正话反说、塑造情绪、价格利益、身份代入、场景激发、剧情引入、国货情怀、背书承诺。
3. 再判断三秒停留技巧：爆点前置、三级分镜式剪辑、静态文字钉、单画面信息密集化、植入槽点、移花接木嫁接泛内容、新闻事件式开头。
4. 最后判断后文是否接住前三秒制造的停留理由；如果前三秒吸引来的人，后面内容没有兑现，要写清楚这是开头失效或中段承接失效。

前三秒输出必须按“证据 -> 家长心理 -> 强弱点”的链条写，不要写成字段翻译。

图片/图文作品不要硬编前三秒，必须改看：

1. 首图/前两张图为什么让用户愿意继续看。
2. 图片顺序有没有形成信息递进，是清单、过程、对比、情绪推进，还是单纯生活记录。
3. 标题和话题有没有补足观看理由。
4. 后续图片有没有回答家长问题；如果只是好看，要写清为什么难以激发评论、收藏或分享。

正确写法：

```text
前三秒先给反面判断：别再买劣质声光电。它打中的不是“我想看看礼物”，而是“我怕又给孩子买一堆没价值的玩具”。桌面实物的作用是告诉家长：后面不是吐槽，会直接给替代方案。
```

错误写法：

```text
前三秒真人出镜，桌面有多款实物，字幕出现六一儿童节礼物清单。
```

凡是写到“真人出镜 / 实物展示 / 孩子画面 / 口播开头”，必须继续回答：

1. 这个开头把哪类家长拉进来了。
2. 她当下怕什么、想确认什么、想解决什么。
3. 开头强在哪里，弱在哪里。
4. 这个强弱会影响点赞、评论、收藏、分享里的哪一项。

中段停留机制也必须按“画面动作 -> 解决的问题 -> 互动动作”写。不要只写“多款实物、孩子实操、场景切换、信息密度高、真实感强”。这些只是素材现象，不是判断。

正确写法：

```text
中段不是靠画面多，而是每个商品后面都接孩子真实使用场景。它在回答家长最关心的三个问题：孩子会不会真用、能不能玩久、是不是比声光电玩具有价值。所以收藏和分享高，是因为家长可以把它当候选清单拿去讨论。
```

错误写法：

```text
中段靠多款实物、孩子户外拍照、孩子骑车、桌面玩具轮换留人。
```

遇到“功能点多 / 信息密度高 / 真实感强 / 有参考价值 / 生活氛围好”这类词，必须立刻改写成具体问题：

| 空泛词 | 必须改成 |
|---|---|
| 功能点多 | 哪个功能会让家长追问什么，比如定位、通话、升级、适合年龄 |
| 信息密度高 | 解决了哪些买前问题，比如素材从哪来、孩子能不能独立用、大人要不要帮 |
| 真实感强 | 它证明了什么，比如孩子真的会用、用了很久还没闲置、不是摆拍 |
| 有参考价值 | 参考什么：候选清单、选购标准、步骤、价格、型号、年龄 |
| 生活氛围好 | 只说明能看，不等于能评论、收藏或分享；要判断有没有具体问题 |

抽样来源：

1. 高互动作品。
2. 中位互动作品。
3. 低互动作品。
4. 强商品信号作品。
5. 强品类连接作品。

每条细看作品至少标注：

| 字段 | 说明 |
|---|---|
| 作品ID | 抖音作品 ID |
| 前三秒 | 开头三秒用什么留人；必须写清画面/字幕打中了什么家长心理，不得只列“真人/实物/孩子画面” |
| 不划走理由 | 判断用户为什么愿意先停一下；必须写成她怕什么、想确认什么、想解决什么 |
| 前三秒文案类型 | 判断属于哪类前三秒文案，可多选但必须写依据 |
| 三秒停留技巧 | 判断用了哪类三秒停留技巧，可多选但必须写依据 |
| 前三秒承接是否接住 | 后文有没有兑现前三秒制造的期待 |
| 四项互动锚点 | 这条视频命中的点赞/评论/收藏/分享高位或低位锚点 |
| S层命中人群 | 这条作品主要打哪个阶段的人：她现在在了解、被种草、比较选择、准备下单，还是只是在围观 |
| 互动数据 | 点赞、评论、收藏、分享、总互动 |
| 四项互动校验结论 | 分别解释点赞、评论、收藏、分享为什么高或低；如果数据低，要说明可能是该博主当前粉丝里这类人群不多，或视频技法没有打透 |
| 媒体核心内容 | 这条作品到底讲了什么；旧结果里如果仍叫“视频核心内容”，渲染时可兼容读取 |
| 口播重点 | 达人嘴上主要在说什么 |
| 画面重点 | 画面展示了什么 |
| 中段停留机制 | 中间回答了什么家长问题；必须从画面动作推到“解决了什么问题”，不得只列多款轮换、场景切换、孩子实操 |
| 分析模式 | 挂车/非挂车，来自 2A 粗筛的公开字段判断 |
| 继续看方式 | 非挂车内容填写，只看入口期待、中段回答、情绪推进、信息递进或画面递进如何让用户继续看 |
| 商品说服方式 | 挂车内容填写，只看商品如何出现、解决哪个购买/选择问题、哪里自然或生硬、是否影响评论/收藏/分享 |
| 内容真实感 | 强/中/弱；必须说明真实感证明了什么，不得只写“真实感强” |
| 品类连接来源 | 标题连接/视频内容连接/评论连接待验证 |
| 可借鉴点 | 对后续合作或评论验证有什么启发 |

2B 输出要写人能看懂的话，重点回答：

1. 这条作品主要打哪个 S层人群。
2. 视频看前三秒用什么把这类人留下来；图文看首图/前两张图和图片顺序用什么让人继续看。
3. 作品里到底在讲什么，不要只复述标题。
4. 如果是非挂车，内容靠什么让用户继续看下去：入口期待、中段回答、情绪推进、信息递进，还是画面递进。
5. 如果是挂车，商品靠什么完成购买/选择说服：出现是否自然、解决什么买前问题、有没有让用户问链接/型号/价格/适用场景。
6. 品类连接是媒体内容本身成立，还是只有标题看起来有关。
7. 点赞、评论、收藏、分享四项互动分别有没有验证这个人群方向：点赞说明认可，评论说明讨论/提问，收藏说明复看/决策参考，分享说明可转发；如果某项低，要解释对应动作为什么没被激发；如果四项整体低于样本中位，要写“内容方向相关，但该博主当前粉丝里对应人群响应不强/人群可能不多”。

S层命中人群不要写成“宝妈/家长/儿童家庭”这种静态身份。要写成：

> 某类人 + 她现在卡在哪一步 + 这条视频帮她解决哪一步。

例子：

| 视频类型 | S层命中人群 |
|---|---|
| 六一礼物清单 | 正在给孩子挑礼物、怕买到没用玩具的家长 |
| 儿童相机长期使用推荐 | 已经想买儿童礼物、正在比较哪款更耐玩更值的家长 |
| 儿童健康/状态管理 | 孩子状态让家长担心、正在找解释和解决办法的家长 |
| 返场/补货/尺码 | 已经有购买兴趣、只差颜色尺码或购买入口的人 |

## 2C 优质内容到营养品的转接预判

2C 在 2B 看完真实媒体内容后执行。它不是判断“这个账号适不适合某个具体产品”，而是判断：

```text
账号已经被互动验证过的优质内容机制，能不能转成儿童营养品相关议题。
```

核心问题不是“内容主题像不像营养品”，而是：

```text
现有优质内容靠什么起量 -> 这个机制能不能换成营养品选择问题 -> 换法是否自然 -> 风险在哪里
```

必须先拆优质内容机制，再判断转接。不能直接写“有儿童内容，所以可以接营养品”。

### 执行落地方式

这层判断单独作为 2C，不能放在 2A 粗筛里提前下结论，也不要把每条视频都写成完整转接方案。执行时按六步走：

1. 先从 2B 样本里找“已验证优质内容”。优先看点赞、评论、收藏、分享任一项明显高于样本中位，且视频内容能解释为什么高的作品。
2. 再拆这条内容的起量机制。不要写“儿童内容”“商品内容”“真实感强”，要写它到底解决了家长什么问题，例如怕买错、需要清单、想确认孩子会不会真用、想知道买前标准。
3. 再判断这个家长问题能不能换成营养品选择问题。能换，写具体议题；不能换，写不能硬转。
4. 再写必须保留的表达方式。营养品转接不是换个品类名，而是保留原来有效的讲法，例如避坑、候选清单、长期观察、孩子日常场景、妈妈经验。
5. 再写风险。风险说明哪种转法会让内容变硬，例如纯成分科普、直接推产品、制造健康恐吓、承诺功效。
6. 最后写评论验证点。评论验证点说明下一步要在评论里看什么问题是否真实存在。

“已验证优质内容”必须同时满足两点：

- 数据上至少有一个互动动作高于样本中位，或在四项高低位抽样中能解释为什么它比同类样本更强。
- 内容上能说清它具体打中了哪类家长、她卡在哪一步、视频帮她解决了哪一步。

如果某条营养/健康相关视频本身互动低，不能因此直接否定营养品方向；更准确的写法是：

```text
这条内容主题和营养健康相关，但互动没有验证。后续不能直接照这个营养健康科普打法走，要先借用账号已经跑出来的高互动结构，再把问题换成营养品选择题。
```

如果 2B 样本里没有任何内容机制被互动验证，不要强行写“可以转”。写：

```text
本轮没有看到可支撑营养品转接的高互动内容机制，暂时只能保留为待验证方向。
```

### 转接判断链条

```text
优质内容样本 -> 起量机制 -> 对应的家长问题 -> 营养品可转接议题 -> 必须保留的表达方式 -> 风险 -> 评论验证点
```

示例：

```text
高智礼物清单 -> 家长怕买错、需要避坑和候选清单 -> 转成“儿童营养品别乱买，先看这几个判断” -> 保留妈妈经验、避坑、选择标准 -> 风险是不能变成纯成分科普或直接推产品 -> 评论里验证家长是否真的在问挑食、长高、免疫、安全和怎么选。
```

### 常见转接路径

| 现有优质内容结构 | 可转成的营养品议题 | 转接重点 | 风险 |
|---|---|---|---|
| 礼物/用品清单 | 营养品选择清单：什么情况需要看营养支持，什么情况先看饮食/作息 | 保留“帮家长做选择”的结构 | 不能写成直接推荐某款产品 |
| 反声光电/避坑 | 乱补/盲买避坑：别看到孩子状态不好就乱买补剂 | 保留反面判断和排雷感 | 不能制造健康恐吓 |
| 常见问题拆解 | 营养品买前问题：适合多大、怎么吃、看什么成分、安全吗 | 把购买前疑问拆清楚 | 不能装专家，必须保留妈妈经验口吻 |
| 长期使用推荐 | 长期观察逻辑：不说立刻见效，只说怎么观察孩子状态 | 强调真实观察、使用边界 | 不能承诺功效 |
| 孩子真实使用 | 生活场景：吃饭、睡前、挑食、运动后、换季 | 让营养问题落在日常场景 | 不能只拍产品或只讲成分 |
| 亲子生活 Vlog | 弱转接，只能做场景铺垫 | 先建立生活处境，再进入问题 | 如果没有具体问题，难以带来评论/收藏/分享 |

### 输出要求

2C 必须输出 `02C_营养品议题转接预判.md` 和 `02C_营养品议题转接预判明细.csv`。这是集中总结，不是逐条视频长篇展开。至少回答：

1. 哪些已验证的优质内容机制可以转。
2. 分别能转成什么儿童营养品议题。
3. 转接时必须保留什么表达方式。
4. 哪些转法风险高，为什么。
5. 如果现有营养/健康相关样本互动低，要写清：不能直接按营养健康垂类打法走，必须借用账号已验证过的内容结构。

`02C_营养品议题转接预判.md` 建议写成这五行，不要写成大段空话；每行 1-2 句话即可：

| 输出行 | 写什么 |
|---|---|
| 已验证的内容结构 | 哪几条高互动内容证明了什么结构有效 |
| 可以转的议题 | 这些结构能换成哪些儿童营养品问题 |
| 不能丢的讲法 | 转接时必须保留避坑、清单、真实观察、妈妈经验还是孩子场景 |
| 高风险转法 | 哪些转法会变成硬广、成分课、功效承诺或健康恐吓 |
| 下一步看评论 | 看评论里有没有真实的营养/健康/育儿焦虑问题 |

控制篇幅：

- 2B 单条视频明细不展开营养品转接。
- 2C 单独输出，不塞进 `02B_媒体内容细看.md` 主体里。
- 如果没有被互动验证的优质机制，这一节写 1 句话即可，不要硬凑转接方案。
- 不要让营养品预判盖过 2B 主线；2B 主线仍然是 S层人群、前三秒/首图、中段承接、口播画面、按挂车/非挂车分开的第五段、真实感和四项互动归因。

正确写法：

```text
这个账号不能直接变成营养健康科普号。更自然的转法，是把营养品写成“家长选择题”：孩子状态不好时，先判断饮食、作息、运动和营养支持分别该怎么看。也就是沿用她已经跑出来的“避坑 + 候选清单 + 孩子真实场景”结构。
```

错误写法：

```text
账号有儿童内容，可以接营养品。
```

## 品类连接母类

| 母类 | 延伸子类 | 看什么 |

| 儿童 | 儿童成长、学习、玩具、儿童用品、儿童情绪、生活习惯 | 内容对象是不是孩子，而不是泛生活 |

不要单独把“睡眠”设为品类连接维度。作品里出现睡眠时，只归入儿童健康、生活习惯或育儿焦虑，不单独开睡眠赛道判断。

## 连接强度判断

判断顺序固定为：

1. 内容对象是谁。
2. 问题是否和孩子/妈妈有关。
3. 是否能自然接到营养健康决策。

| 强度 | 判断标准 | 示例 |
|---|---|---|
| 强连接 | 内容主体是孩子/妈妈，且问题直接指向营养、健康、喂养、发育或育儿焦虑 | 挑食、长高、免疫、肠胃、孩子状态不好、喂养困难 |
| 中连接 | 内容主体是孩子/家庭，但不直接指向营养健康 | 儿童玩具、亲子日常、学习习惯、儿童用品 |
| 弱连接 | 只是偶尔提到孩子或妈妈，核心仍是泛生活/个人表达 | 妈妈穿搭、家庭日常、情绪吐槽 |
| 无连接 | 和母婴、儿童、营养、健康、育儿焦虑无关 | 泛娱乐、成人消费、无孩子场景的生活内容 |

## 商品内容信号判断

第二项很难判断一条内容是不是广告、合作、带货或收取推广费用，所以不要直接下“商业内容”结论。本项只判断商品内容信号：这条内容是否在推荐商品、帮粉丝选东西，或做种草。

| 档位 | 判断标准 |
|---|---|
| 强商品信号 | 标题、话题、口播或画面明确出现品牌词、商品词、具体产品推荐、使用体验推荐、价格优惠、购买引导、商品锚点、店铺/橱窗、合作/广告/推广声明 |
| 弱商品信号 | 有产品展示、开箱、测评、对比、好物语气或消费场景，但无法确认是否在推荐具体商品 |
| 无明显商品信号 | 主要是日常、观点、育儿经验、情绪表达、生活记录，没有具体商品、品牌、购买引导或种草意图 |

商品仲裁必须看标题里的 `#` 话题。`#` 后面如果出现品牌词、商品词、品类词、活动词，优先作为商品信号证据记录。

挂车判断也要做，但只能基于公开接口是否返回有效字段：

| 结果 | 判断 |
|---|---|
| 是 | `anchors`、商品锚点、店铺/橱窗、电商字段等公开字段有有效返回 |
| 未发现 | 原始接口未返回有效挂车字段；只能说明“公开字段未发现”，不能等于“确定没有挂车” |
| 未知 | 缺少 `raw.json` 或原始接口数据，无法判断 |

示例：

| 内容 | 判断 |
|---|---|
| “这款儿童相机我家用了三年，六一推荐” | 强商品信号：具体商品推荐/种草 |
| “儿童AI照相机，第三年推荐了” | 强商品信号：具体商品词 + 推荐语气 |
| “#某品牌 #儿童相机 #六一礼物” | 强商品信号：话题出现品牌/商品/品类词 |
| “几个高智礼物分享 / 礼物清单 / #儿童节礼物” | 强商品信号：礼物推荐内容 |
| “孩子最近挑食，我试了几个办法” | 无明显商品信号：经验分享，没有具体商品 |

判断重点不是猜达人有没有收广告费，而是判断这条内容是不是在推荐东西、帮粉丝选东西。

## 第二项 MD 输出结构

`02A_内容资产粗筛.md` 是 2A 粗筛报告，必须在标题或样本口径里写明“本轮未看真实媒体内容，只做标题/话题/公开字段粗筛”。

2A 必须按样本池生成：有普通作品才生成 `02A_内容资产粗筛.md`，有挂车作品才生成 `02A_挂车作品内容资产粗筛.md`。全挂车时不要生成普通 2A；无挂车时不要生成挂车 2A；混合时两套分开生成。

`02B_媒体内容细看.md` 是 2B 媒体内容细看报告，必须基于真实媒体内容输出 S层命中人群、视频前三秒或图文首图/图片顺序、口播/字幕/标题、画面、按挂车/非挂车分开的第五段和真实感判断。

Markdown 正文只写判断，不单独展示本地视频路径、图片路径、抽帧图路径或完整运行目录，也不单独输出“口播/字幕依据”段落。这些字段保留在明细 CSV 用于追溯。正文需要引用字幕或画面时，必须把它嵌入到判断里，说明它证明了什么、打中了哪类人、影响哪项互动。

正文排版必须让人读得下去：`入口类型 / 停留技巧 / 后续承接` 分行写；`继续看方式 / 内容真实感` 或 `商品说服方式 / 内容真实感` 分行写，不要用一行分号串起来。

2B 正文固定为深拆模式，不得退回字段清单式报告。每条作品固定顺序：

1. S层命中人群：人群画像 + 她卡在哪一步 + 内容解决哪一步。
2. 前三秒/首图：为什么不划走。视频看前三秒，图文看首图/前两张图；必须写入口证据、用户心理、入口类型、停留技巧和后文承接。
3. 中段承接分析：中段是否接住入口期待，强在哪里，断在哪里。
4. 标题 / 字幕 / 画面关系：标题、字幕、口播和画面是否形成承诺与证据闭环。
5. 第五段按分析模式二选一：
   - 非挂车：输出继续看方式，只写怎么让用户继续看下去，比如入口期待、中段回答、情绪推进、信息递进或画面递进；不要写商品/购买说服。
   - 挂车：输出商品说服方式，只写商品如何出现、解决哪个购买/选择问题、哪里自然或生硬、是否影响评论/收藏/分享；不要写非商品继续看。
6. 真实感：真实感是否帮助用户继续看。
7. 四项互动归因：点赞、评论、收藏、分享分别解释高低原因。
8. 品类连接和评论验证：连接来源和下一步评论验证点。

多条样本时，文末必须输出多样本对比总结，比较 S层精准度、分析模式、留人入口、中段承接、继续看/商品说服方式和真实感。

`02C_营养品议题转接预判.md` 是 2C 优质内容到营养品的转接预判报告，必须基于 2A + 2B 结果输出，不替代 2B 媒体拆解。

2. 内容主题分布

## S层和互动数据校验

S层判断分两步，不要混在一起：

1. 先看标题、真实媒体内容、视频前三秒或图文首图/图片顺序、口播/字幕/标题和画面，判断这条内容打的是哪类人、她卡在哪一步。
2. 再分别看点赞、评论、收藏、分享，判断这类人群在该账号粉丝里有没有响应，以及响应具体表现在哪个动作上。

2B 的判断链条必须固定为：

```text
视频证据 -> 打中的家长问题 -> 这类人是否在账号粉丝里有响应 -> 响应体现在哪个互动动作
```

示例：

```text
孩子真实使用相机 -> 家长想确认孩子会不会真的拿出去用 -> 点赞高说明认可“不是摆拍/不是新鲜劲” -> 但没有给年龄、型号、替代品标准，所以评论、收藏、分享没有跟着放大。
```

禁止写成：

```text
孩子真实使用画面，真实感强，所以点赞高。
```

互动数据的用法：

| 数据 | 说明 |
|---|---|
| 点赞 | 这条内容是否被认可、是否有基本共鸣 |
| 评论 | 有没有真实讨论、提问、争议或经验交换 |
| 收藏 | 有没有复看价值、购买/选择参考价值 |
| 分享 | 是否适合转发给家人、朋友或同类家长 |

2B 禁止只写“总互动高/低”。必须按四项互动分别写：

```text
点赞为什么高/低？
评论为什么高/低？
收藏为什么高/低？
分享为什么高/低？
```

并把原因落到视频技法上：

```text
前三秒技法
中段停留机制
信息到底解决了哪个问题
真实使用画面证明了什么
选择标准
未解问题
可转发对象
```

四项互动归因必须写“为什么这个动作被激发/没有被激发”：

| 互动 | 不能只写 | 必须写成 |
|---|---|---|
| 点赞 | 有真实使用场景，所以点赞高 | 真实使用证明孩子确实会用，降低“买来闲置”的担心，所以点赞高 |
| 评论 | 功能点多，所以评论高 | 抛出了定位、通话、升级这些未解问题，用户才有追问入口 |
| 收藏 | 有参考价值，所以收藏高 | 给了候选清单/选购标准/步骤，家长有回头核对的理由 |
| 分享 | 适合转发，所以分享高 | 能拿给共同决策人讨论“买不买/买哪款/值不值” |

如果内容上打“孩子状态担心/育儿焦虑/儿童用品选择/营养健康决策”这类人群，但点赞、评论、收藏、分享整体低于样本中位，结论要写成：

> 内容上打的是 XX 人群，但互动数据没有验证这个方向。初判该博主当前粉丝里这类人群不多，或这条内容没有把这类人打透，不能把它当账号稳定人群方向。

如果内容上打某类人群，且收藏、分享、评论明显高于中位，可以写：

> 内容上打的是 XX 人群，互动数据也支持这个判断，尤其收藏/分享/评论说明用户不只是点赞，而是在把它当选择参考或讨论入口。

## 初步判断写法

要直接给结论，不要只写“需要进一步判断”。

可以写：

> 该账号内容资产主要集中在儿童用品/亲子日常，和“儿童”有中连接，但与营养、健康、育儿焦虑的直接连接不足。当前可以继续看评论信任盘，但合作方向更适合从儿童使用场景切入，不适合直接打营养功效。

也可以写：

> 该账号内容资产和母婴营养品有强连接：内容主体多为孩子和妈妈，主题集中在喂养、挑食、健康管理和妈妈选择困难。建议进入评论信任盘，重点看粉丝是否真实表达喂养焦虑和产品接受度。

## 边界

1. 标题只能做粗筛，不能替代真实媒体内容判断。
2. 商品内容信号不等于广告、合作、带货或商业转化能力。
3. 不要为了贴合具体产品强行扩大连接范围；没有营养健康决策空间时，要明确写弱连接或无连接。
4. 第二项不默认写具体产品怎么接；具体产品承接放到后续产品承接阶段。
